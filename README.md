# DevAgent Local

Локальный агент для создания и последовательного редактирования небольших проектов через Ollama.

## Текущее состояние

Реализовано новое модульное ядро:

- отдельные проекты в `projects/<slug>`;
- постоянные сессии и сообщения в SQLite (`.devagent/devagent.db`);
- безопасная обработка путей без выхода за пределы проекта;
- атомарная запись UTF-8 файлов;
- автоматические контрольные копии перед перезаписью;
- получение списка моделей непосредственно из Ollama;
- генерация каждого файла отдельным запросом, чтобы не обрывать весь проект одним лимитом токенов;
- загрузка существующих файлов в контекст при последующих изменениях.
- проверка HTML на Markdown-ограждения, незакрытый документ и отсутствующие локальные CSS/JS/изображения.
- агентный цикл с инструментами `list_files`, `read_file`, `search_text`, `write_file`, `replace_text`, `delete_file` и `validate_project`;
- совместимость как с нативным tool calling Ollama, так и с JSON-вызовами инструментов старых шаблонов моделей;
- раскрываемый журнал действий агента в истории чата.
- безопасный `run_project_checks`: JavaScript проверяется через изолированный `node --check`, Python компилируется без запуска, CSS проверяется на базовые синтаксические дефекты;
- таймаут и allowlist для внутренних проверочных команд без доступа модели к произвольному shell.
- обязательный `browser_check` после статических тестов: headless Chromium открывает сайт на desktop 1440×900 и mobile 390×844;
- обнаружение ошибок JavaScript в консоли, незагруженных CSS/JS, почти пустой страницы и горизонтального переполнения;
- скриншоты каждого браузерного прогона в `.devagent/artifacts`, доступные прямо из истории чата.
- фиксированный React 19 + Vite 8 runtime внутри Docker-образа без выполнения `npm run`, install-скриптов и пользовательских `vite.config.*`;
- временный Vite dev-сервер для проверки JSX/TSX и production-сборка успешного проекта в `.devagent/previews`;
- кнопка «Открыть сайт» показывает опубликованную Vite-сборку, а не необработанный JSX;
- DOM-проверки подписей полей, доступных названий кнопок/ссылок и submit-кнопок форм.
- автоматический выбор модели: быстрый `qwen2.5-coder:7b` для обычных задач и `qwen2.5-coder:32b` для запросов на современный/premium-дизайн; фактически использованная модель показывается в сообщении.

Старый прототип и созданные им примеры сохранены в каталоге `legacy/`. Новый сервер запускается командой `python -m devagent` внутри контейнера.

## Запуск

1. Запустите Ollama и Docker Desktop.
2. Дважды щёлкните `start_agent.bat`.
3. Дождитесь сообщения `DevAgent is running`. Браузер откроется автоматически.

Папка больше не зашита в скрипт: используется директория, в которой находится `start_agent.bat`.

## Тесты

После сборки образа:

```powershell
docker run --rm devagent-local python -m unittest discover -s tests -v
```

## Следующий технический шаг

Расширить браузерную проверку сценариями взаимодействия с формами, кнопками, модальными окнами и навигацией. Старый `BuilderService` сохранён временно для сравнения, но HTTP API уже использует `AgentService`.
